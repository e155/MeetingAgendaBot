# 🤝 Meeting Bot v2

Бот для ведения митингов в Telegram-группах.  
**Все команды выполняются в личном диалоге (ЛС) с ботом — группа остаётся чистой.**  
В группу публикуются только пункты повестки, решения и итоговый отчёт.

---

## 🚀 Установка

```bash
pip install -r requirements.txt
```

## ⚙️ Настройка

Нужно задать две переменные окружения:

```bash
export BOT_TOKEN="токен_от_BotFather"
export GROUP_ID="-1001234567890"   # ID вашей группы
```

### Как узнать GROUP_ID?
1. Добавьте бота [@userinfobot](https://t.me/userinfobot) в группу
2. Отправьте любое сообщение — он покажет ID группы (отрицательное число)
3. Или напишите `/start` боту в группе и посмотрите логи

### Настройка BotFather
1. `/setprivacy` → ваш бот → **Disable** (чтобы бот мог видеть сообщения)
2. Добавьте бота в группу как администратора (минимум — право писать сообщения)

## 🏃 Запуск

```bash
python bot.py
```

## 📋 Команды (все в ЛС с ботом)

| Команда | Описание |
|---------|----------|
| `/start` | Регистрация, привязка к группе |
| `/agenda` | Добавить пункт повестки (2 шага: название → детали) |
| `/shagenda` | Показать текущую повестку |
| `/task` | Создать задачу (4 шага: название → детали → ответственный → срок) |
| `/tasks` | Открытые задачи с кнопками «Выполнено» / «Передать другому» |
| `/history` | История закрытых задач |
| `/newmeeting [Название]` | Начать митинг — опубликовать повестку в группе |
| `/decision` | Зафиксировать решение по текущему пункту повестки |
| `/pending` | Отложить пункт до следующего митинга |
| `/summary` | Завершить митинг и вывести отчёт в группу |

---

## 🔄 Сценарий работы

### До митинга (в ЛС):
1. Участники пишут `/task` — создают задачи
2. Участники пишут `/agenda` — добавляют пункты повестки

### Во время митинга:
1. Ведущий пишет `/newmeeting Название` → бот публикует повестку в группе
2. По каждому пункту — кнопка **«Следующий пункт»** переключает повестку в группе
3. В ЛС ведущий пишет `/decision` → вводит решение → выбирает:
   - **Done** — решение принято, публикуется в группу
   - **ToDo** — создаётся задача + публикуется в группу
4. `/pending` — пункт откладывается до следующего митинга
5. `/summary` — отчёт публикуется в группу, митинг закрывается

### Задачи:
- `/tasks` — список с кнопками
- **«Передать другому»** — 4-шаговый диалог переназначения
- `/history` — архив выполненных задач
