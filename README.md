# Delphi IT Meeting Bot

Telegram-бот для ведения митингов в корпоративных группах.  
Все личные команды — в **ЛС с ботом**. В группу публикуются только повестка, решения и PDF-протокол.

---

## Быстрый старт

### 1. Установка

```bash
git clone <repo> && cd mbot5
python3 -m venv venv && source venv/bin/activate
pip install -r requirements.txt
apt-get install -y fonts-dejavu-core
```

### 2. Файл `.env`

```env
BOT_TOKEN=YOUR_BOT_TOKEN          # токен от @BotFather
GROUP_ID=-1009999999999    # ID группы (узнать: написать /chatid в группе)
ADMIN_IDS=123456789        # user_id администраторов через запятую
BACKUP_OPERATOR=admin      # "admin" — только ADMIN_IDS, "all" — все пользователи
PDF_TITLE=Delphi IT Meeting Agenda  # заголовок PDF-протокола
```

### 3. Права бота в группе

- `@BotFather` → `/setprivacy` → **Disable**
- Группа → **Administrators** → бот → включить **Delete Messages + Send Messages**

### 4. Запуск

```bash
python bot.py
```

### 5. Автозапуск (systemd)

```ini
[Unit]
Description=Delphi IT Meeting Bot
After=network.target

[Service]
WorkingDirectory=/root/mbot5
ExecStart=/root/mbot5/venv/bin/python bot.py
EnvironmentFile=/root/mbot5/.env
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

```bash
systemctl enable meeting-bot && systemctl start meeting-bot
```

## Команды

### Личный диалог (ЛС с ботом)

| Команда | Описание |
|---------|----------|
| `/start` | Регистрация, список команд |
| `/agenda` | Добавить пункт повестки (2 шага: название → детали) |
| `/shagenda` | Повестка: нерешённые с прошлого митинга + очередь + текущий митинг |
| `/task` | Создать задачу (4 шага: название → детали → ответственный → срок) |
| `/tasks` | Открытые задачи — кнопки «Закрыть» (с подтверждением) и «Edit» |
| `/history` | Последние 20 закрытых задач с авторами |
| `/backup` | Скачать резервную копию базы данных |
| `/restore` | Восстановить базу из файла (с подтверждением и авторестартом) |

### В группе (только организатор митинга)

| Команда | Описание |
|---------|----------|
| `/newmeeting [Название]` | Начать митинг, опубликовать повестку |
| `/next` | Следующий пункт циклически (не закрывает текущий) |
| `/decision` | Зафиксировать решение: **Done** (закрыт) или **ToDo** (создать задачу) |
| `/pending` | Отложить пункт до следующего митинга |
| `/summary` | Завершить митинг — итоги в группу + PDF-протокол |

> `/newmeeting` может запустить любой участник группы.

---

## Сценарий митинга

**До митинга** (в ЛС):
- участники добавляют пункты через `/agenda` — попадают в очередь
- создают задачи через `/task`, выбирают ответственного из кнопок

**Запуск** (в группе):
```
/newmeeting Q1 Planning
```
Бот публикует повестку и первый пункт с подсказками.

**Ход митинга** (в группе, организатор):
```
/next      → следующий пункт (циклически, без закрытия)
/decision  → решение → Done или ToDo → следующий пункт автоматически
/pending   → отложить → следующий пункт автоматически
/summary   → итоги + PDF
```
Все вопросы диалога и ответы организатора удаляются — остаётся только итоговое сообщение.

**Следующий митинг**: при `/newmeeting` автоматически подтягиваются нерешённые пункты прошлого митинга и очередь.

## PDF-протокол

Генерируется при `/summary`. Разделы:

| Раздел | Содержимое |
|--------|-----------|
| **Agenda** | Все пункты со статусами (Closed / Deferred / Discussed / Not reviewed) |
| **Decisions** | Тип (Done/ToDo), текст, ответственный, ссылка на пункт |
| **Deferred** | Отложенные пункты с заметками |
| **Open Tasks** | Задачи с ответственным, сроком, автором и редактором |

Шрифт: DejaVu Sans (кириллица). Цвет: `#3b8ad1`.

---

## Задачи

- Создаются через `/task` в ЛС — 4 шага с выбором ответственного из кнопок
- Редактируются через кнопку **Edit** в `/tasks`
- Закрываются с подтверждением «Да/Нет»
- `/history` — последние 20 закрытых, с автором, редактором и тем кто закрыл

---

## Резервное копирование

```
/backup  → получаете файл meetings_backup_YYYYMMDD_HHMMSS.db
/restore → отправляете .db файл → подтверждение → применить + рестарт
```

Перед применением бекапа текущая база сохраняется как `meetings.db.bak`.

## База данных (SQLite)

| Таблица | Содержимое |
|---------|-----------|
| `users` | Участники (автосохранение при любом сообщении в группе) |
| `meetings` | Митинги, организатор, статус, текущий индекс повестки |
| `agenda_items` | Пункты повестки с автором и статусом |
| `pending_agenda` | Очередь пунктов до старта митинга |
| `decisions` | Решения (done/todo), ответственный, пункт повестки |
| `tasks` | Задачи: создатель, редактор, исполнитель, закрывший |
| `pending_items` | Отложенные пункты для переноса на следующий митинг |
